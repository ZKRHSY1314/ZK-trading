# 产品需求文档

## 1. 产品定位

本系统是一个 A股 AI 交易驾驶舱。第一版不追求实盘自动下单，而是把全A机会发现、候选池监控、规则决策、AI解释、模拟交易和复盘学习串成闭环。

系统服务对象是有明确个人交易经验和战法体系的操作者。AI 的角色不是替代交易者，而是帮助交易者减少买早、买高、乱补仓和未执行计划等重复错误。

## 2. 第一版目标

1. 集合竞价和开盘阶段全A扫描，识别候选股票。
2. 将候选池分为强候选、观察候选、剔除名单。
3. 盘中以 1 分钟频率监控候选池。
4. 使用可配置规则引擎承载灯盏策略和后续战法。
5. 严格模拟 A股交易规则，记录模拟交易流水。
6. 在控制台展示买入、卖出、止损、主力出货等提醒。
7. 支持模拟交易按钮，实盘按钮显示但禁用。
8. AI 自动调整策略权重，但必须经过回测/模拟指标验证。

## 3. 核心用户流程

### 3.1 盘前/集合竞价

1. 系统拉取全A基础行情和昨日数据。
2. 规则引擎计算候选评分。
3. 系统生成强候选、观察候选、剔除名单。
4. 控制台展示进入候选池的原因和违反红线的原因。

### 3.2 盘中

1. 系统每分钟刷新候选池行情。
2. 灯盏策略、买卖战法和风控规则依次判断。
3. AI 对命中信号生成解释和相似案例参考。
4. 用户可点击模拟买入、模拟卖出、忽略、加入剔除名单。
5. 所有提醒和用户操作写入事件日志。

### 3.3 收盘后

1. 系统计算模拟账户收益、回撤、胜率、盈亏比。
2. 系统生成当日复盘报告。
3. AI 根据交易表现提出规则权重调整。
4. 权重调整必须通过回测/模拟验证后才能进入下一轮模拟。

## 4. 决策优先级

1. 交易铁律红线：命中红线时禁止给出积极买入建议。
2. 风控：仓位、止损、T+1、涨跌停、最小交易单位。
3. 策略规则：灯盏策略、隔夜单、打板、均线支撑、分批止盈等。
4. 案例相似度：成功/失败案例作为解释和风险提示。
5. AI解释：只负责解释、总结、调权建议，不越过规则和风控。

## 5. 功能范围

### 第一版包含

- 全A扫描入口。
- 候选池分层。
- 规则配置文件。
- 规则命中解释。
- 灯盏策略骨架。
- 严格模拟交易骨架。
- 回测指标骨架。
- AI模型可插拔接口。
- 权重优化器骨架。
- 本地网页控制台骨架。

### 第一版不包含

- 实盘自动下单。
- 真实券商接口。
- 多账户资金同步。
- 高频交易。
- 保证收益的预测模型。
- 未验证的自动策略上线。

## 6. 验收标准

1. 可以启动本地后端并看到健康检查。
2. 可以通过配置文件定义规则、权重和阈值。
3. 可以产生候选股票分层结果的数据结构。
4. 可以创建模拟订单，并按 A股约束做基础校验。
5. 可以输出收益、最大回撤、胜率、盈亏比等指标。
6. AI权重调整必须输出调整理由和验证结果。
7. 实盘相关能力在界面和接口层保持禁用状态。
