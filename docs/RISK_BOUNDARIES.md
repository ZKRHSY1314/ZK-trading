# 风控和权限边界

## 1. 资金安全边界

第一版禁止实盘自动下单。系统只能做模拟交易、提醒、复盘、策略权重实验。

任何真实券商接口都必须在后续版本单独设计，并满足：

1. 人工确认开关。
2. 单日最大亏损限制。
3. 单票最大仓位限制。
4. 熔断暂停机制。
5. 完整审计日志。
6. 可一键切回只读监控模式。

## 2. AI权限边界

AI 可以：

- 解释规则命中原因。
- 总结相似成功/失败案例。
- 提出复盘建议。
- 调整模拟盘策略权重。
- 发起回测验证请求。

AI 不可以：

- 绕过交易铁律。
- 绕过风控规则。
- 直接修改实盘权限。
- 直接发出真实下单请求。
- 在未验证时启用新权重。

## 3. 策略上线边界

任何策略权重更新必须满足全部条件：

1. 模拟收益改善。
2. 最大回撤不恶化。
3. 胜率不恶化。
4. 盈亏比不恶化。
5. 交易次数不能低到没有统计意义。

若指标冲突，默认拒绝自动更新，只保留为人工复盘建议。

## 4. A股模拟约束

模拟交易必须至少包含：

- T+1 卖出限制。
- 涨跌停无法成交约束。
- 集合竞价成交规则占位。
- 买卖最小单位 100 股。
- 手续费。
- 印花税。
- 滑点。
- 停牌和无行情处理。

## 5. 提醒文案边界

系统提醒应使用“建议”“观察”“风险提示”等措辞，不使用“必涨”“稳赚”“无风险”等表述。

每条买入提醒必须同时展示：

1. 命中规则。
2. 违反红线检查结果。
3. 止损位。
4. 仓位建议。
5. 相似失败案例。
